# blockchain/network/

Hyperledger Fabric network configuration files.

## What Goes Here

| File / Folder | Purpose |
|---|---|
| `configtx.yaml` | Channel configuration — organisations, policies, orderer settings |
| `crypto-config.yaml` | Cryptogen input — defines peers and orderers for certificate generation |
| `crypto-config/` | Generated TLS and MSP certificates (generated, not committed to git) |
| `channel-artifacts/` | Generated genesis block and channel transaction (generated, not committed) |
| `docker-compose-fabric.yml` | Docker Compose for Fabric peers and orderer |

## Organisations

BSC runs three organisations on the main channel (`bsc-channel`):

| Organisation | MSP ID | Role |
|---|---|---|
| IT Department | `ITDeptMSP` | Income tax data writer |
| Property Registrar | `RegistrarMSP` | Property registration writer |
| Ministry of Corporate Affairs | `MCAMSP` | Company director data writer |

## Network Topology

```
Orderer (Solo, Phase 1 → Raft, Phase 4)
    └── bsc-channel
            ├── peer0.itdept.bsc.gov (Mumbai DC)
            ├── peer0.registrar.bsc.gov (Delhi DC)
            └── peer0.mca.bsc.gov (Bengaluru DC)
```

## Generated Files — Not Committed

The following are generated by scripts and must NOT be committed to git:

- `crypto-config/` — contains private keys
- `channel-artifacts/` — environment-specific genesis blocks

These are listed in `.gitignore`.

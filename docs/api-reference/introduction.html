API Reference

The BSC API gateway exposes 22 REST endpoints for interacting with the Hyperledger Fabric blockchain. All endpoints are served at http://localhost:4000 in development.

Authentication​

All endpoints except /health, POST /auth/login, POST /auth/guest, and POST /auth/totp/verify require a valid JWT Bearer token:

Authorization: Bearer <token>

Get a token:

curl -s -X POST http://localhost:4000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"identifier":"rajesh.kumar@itdept.bsc.gov","password":"password","role":"IT_DEPT"}'

Response:

{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": { "sub": "abcdef...", "role": "IT_DEPT", "name": "Rajesh Kumar" }
  }
}

Guest token (PUBLIC role, no credentials):

curl -s -X POST http://localhost:4000/auth/guest

Conventions​

Monetary Values​

All monetary values are in paisa as a 64-bit integer. Never floats.

1 INR = 100 paisa
₹50,000 = 5,000,000
₹1 crore = 100,000,000

Identifiers​

All citizen, property, officer, and log identifiers are 64-character lowercase hex strings (SHA-256 format):

a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2

Response Envelope​

All responses follow:

{
  "success": true,
  "data": { ... },
  "txId": "fabric_transaction_id"
}

Errors:

{
  "success": false,
  "error": "Human-readable error message"
}

Role-to-Endpoint Summary​

TagEndpointsRoles
auth/auth/login, /auth/guest, /auth/totp/*Public / ADMIN
citizens/citizens, /citizens/:hash, /citizens/:hash/flags, etc.varies — see each endpoint
properties/properties, /properties/:id, /properties/:id/freezeIT_DEPT, COURT, all roles for reads
flags/flags, /flags/:id, /flags/manualIT_DEPT, ED, CBI, ADMIN
zkp/zkp/:hash/prove, /zkp/:hash/claimsAll authenticated roles
admin/admin/health, /admin/stats, /admin/officers, /admin/permissionsADMIN only

Rate Limits​

  • Global: 200 requests per 15 minutes per IP
  • /auth/*: 20 requests per 15 minutes per IP

Exceeding the limit returns 429 Too Many Requests.

Download OpenAPI Spec​

The full OpenAPI 3.1 spec is available at:

  • In this repo: docs/api/openapi.yaml
  • Raw download: openapi.yaml

Import into Postman, Insomnia, or any OpenAPI-compatible client.
